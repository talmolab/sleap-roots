    • sleap-roots circumnutation analyze + sleap_roots.circumnutation.adapters.series_to_inputs — the user-facing circumnutation CLI and the Series → CircumnutationInputs adapter (PR #17 of the circumnutation program; OpenSpec change add-circumnutation-cli). Two NEWLY created modules adapters + cli (implementation count 12 → 14 by ADDITION — two modules in one PR, a first for the program; stub count UNCHANGED at 1, only parametric/PR #11 remains). One analyze command composes the whole pipeline on one .slp: Series.load → series_to_inputs → compute_traits → write per-plant CSV → aggregate_by_genotype → write per-genotype CSV → save_plots. adapters.series_to_inputs(series, *, cadence_s, sample_uid, series_name=None, timepoint=None, plate_id=None, genotype=None, treatment=None, r_px=None, run_id=None) -> (CircumnutationInputs, identity_provenance) — the single click-free, independently-testable bridge into the pure core, formalizing the former test-only _load_plate001_inputs(): get_tracked_tips() → prefix-anchored ASCII [0-9]+ track_id coercion (strips one leading "track_", then the remainder must be unsigned ASCII digits — raises ValueError naming the offender rather than let int() silently mis-coerce "track_-1"/"track_1_2"→12/"track_ 1"/Unicode digits; "track_track_1" raises, not the global-.replace corruption to 1); plant_id = track_id; CSV-as-source / flag-as-override metadata precedence where a blank value (None/empty/whitespace) on EITHER the flag or the CSV cell resolves to absent/NaN (so --genotype "" or a whitespace cell never forms a degenerate empty-string group); timepoint read as the raw get_metadata cell and str-normalized (object dtype, bypassing the float-coercing Series.timepoint property); the six string identity columns forced to object dtype even when all-NaN. Returns an identity-provenance dict {metadata_csv_path (resolved-abs or None), metadata_csv_sha256 (whole-file hex digest or None), identity_source} where identity_source is total over the six adapter-populated fields (series/sample_uid/timepoint/plate_id/genotype/treatment; plant_id/track_id excluded — derived from the .slp) mapping each to "flag"/"metadata_csv"/"default" (the series filename-stem default) / "absent" (NaN). Logs a WARNING when --metadata-csv is supplied but sample_uid matches no plant_qr_code row (the join would otherwise silently ship all-NaN metadata), and an INFO override-notice only when a non-blank flag shadows a real, different CSV value. cli — a circumnutation click group + analyze command registered on the root CLI via main.add_command(circumnutation) (mirroring viewer); --cadence-s and --sample-uid required (--sample-uid is the stable QR/sample id + metadata-CSV join key, never silently synthesized, per CC-4); --output-dir defaults to ./<series-name>_circumnutation/. Gated per-genotype aggregation (--no-aggregate): when aggregation is on (default) and genotype is unresolved, analyze hard-errors (exit 1, before any output is written) rather than emit a vacuous (NaN,NaN,NaN) per-genotype CSV; --no-aggregate opts out (per-plant + plots only, no genotype required). --no-plots gates the matplotlib import entirely; otherwise matplotlib.use("Agg", force=True) runs before plotting is imported (headless; force=True is load-bearing because Series imports pyplot at module top). Error contract mirrors viewer/cli.py: click handles usage errors (exit 2 — missing required option, nonexistent .slp, non-numeric cadence); the adapter normalizes its raisers to ValueError; the command body is wrapped in one try/except (ValueError, FileNotFoundError) → ClickException (exit 1, clean one-line message, no traceback); no broad catch-all (genuine bugs surface tracebacks). -v/-vv → WARNING/INFO/DEBUG to stderr (stdout reserved for the result summary), restored in a finally so the CLI's global logging config doesn't leak; the CLI also quiets matplotlib's cosmetic font_manager findfont fallback WARNINGs (the scaleogram LogNorm colorbar's mathtext) to ERROR. CC-3 pure-pixel: no --px-per-mm; --help states outputs are pixel-native (pointing to convert_to_mm), documents the image y-down convention, and notes timepoint is a string label. Output tree: a top-level <output-dir>/run_metadata.json (the canonical run provenance, and the target of save_plots' hardcoded run_metadata_ref: "../run_metadata.json") + per_plant/ (CSV + .units.json + run_metadata.json) + per_genotype/ (only when aggregating) + plots/ (only when plotting) — all three run_metadata.json byte-identical (the CLI assembles provenance ONCE via gather_run_metadata and writes via write_per_plant_csv/write_per_genotype_csv directly, NOT CircumnutationPipeline.save(), so the identity provenance is complete and the sidecars share one timestamp). The distinct subdirs sidestep #238 (each _io writer's fixed-name run_metadata.json lives in its own directory) without changing the _io contract — referenced, not closed. _io.gather_run_metadata gains optional metadata_csv_path / metadata_csv_sha256 / identity_source (appended after R_px; backward-compatible — existing callers write null). Tested via click.testing.CliRunner (exit codes, validation, error paths, gated aggregation, --no-plots, -v/-vv, provenance, logging-restore) on a synthetic tracked-.slp round-trip (PIL TIFF → sio.save_slp → real Series.load → get_tracked_tips, incl. the predicted-vs-user dedup branch) carrying non-LFS coverage, plus a skipif-guarded real plate-001 end-to-end (6 plants → genotype=Nipponbare/treatment=MOCK from the metadata CSV; no pixel baselines), and verified by running the actual installed console script on plate-001 (entry point, fresh-process Agg, real exit codes 0/1/2, the on-disk tree + byte-identical sidecars + resolving plots run_metadata_ref). No new constants (_CONSTANTS_VERSION stays 6); no new dependency (click already present). Two spec scenarios reconciled during TDD (interior track_track_1 raises; the unreachable "partial genotype" scenario removed — genotype is filled uniformly per run). Reviewed across brainstorm (D1–D8) + a design critical-review + 3 /openspec-review rounds (each finding a real issue — round 2 caught a BLOCKING dangling plots-provenance pointer from the subdir split, fixed via the top-level run_metadata.json) + 2 /review-pr self-review rounds (pre-PR + post-PR, surfacing the blank-genotype gate gap, the strict-track_id Unicode hole, the logging leak, and the findfont noise — all reconciled). Graduates plotting stub → implementation (implementation count 11 → 12, stub count 2 → 1 — only parametric/PR #11 remains a stub). Four public callables: scaleogram(scaleogram_result, out_path, *, ridge_result=None) -> Path (polymorphic over the temporal ScaleogramResult and spatial SpatialScaleogramResult; power |C|² heatmap on a LogNorm color scale with explicit n+1 cell edges, cone-of-influence dimming, optional ridge overlay faded inside the COI; spatial wavelength axis labeled uncalibrated pywt convention; raises TypeError on an unsupported scaleogram type or a mismatched ridge_result), trail_overlay(midline_result, out_path) -> Path (the smoothed tip path as a per-segment LineCollection colored by the signed curvature_px_inv on a diverging colormap symmetric about 0 with limits ±the 98th percentile of |κ|; NaN-κ segments via a copied colormap's set_bad; image-y-down; extend="both" colorbar), plate_panel(midline_results, out_path) -> Path (a 2×3 grid sharing one κ normalization + one colorbar; raises ValueError for > 6 plants), and save_plots(inputs, out_dir, *, constants=None, enabled=True) -> list[Path] (re-derives the per-plant Results by invoking the same tier helpers the analysis uses — nutation._select_signal + the nutation guards for Tier 1; frame-sort + finite-tip mask + midline.reconstruct → spatial_cwt.resample_curvature → compute_scaleogram/extract_ridge for Tier 3 — so the plotted signal equals the analyzed input). Deviation: the stub-promised L_gz arc-length marker is OMITTED (blocked on #230, not computed in the pipeline) and added by a follow-up PR. Reviewed across brainstorm (D1–D8) + a design critical-review + 3 /openspec-review rounds (each of the first two surfacing a real BLOCKING bug — the _select_signal re-derivation crash on NaN tips, and the non-constructible sidecar back-reference — reconciled before implementation). Follow-up #241 (expose per-plant Result artifacts as MCP-compatible serializable views, removing the plotting recompute). (parent epic #197; tracking issue #242.)
    • sleap_roots.circumnutation.aggregation.aggregate_by_genotype + sleap_roots.circumnutation._io.write_per_genotype_csv / read_per_genotype_csv — per-genotype aggregation layer (PR #15 of the circumnutation program; OpenSpec change add-circumnutation-per-genotype-aggregation). The first post-pipeline aggregation layer, per theory.md §7.7. NEWLY created (never a stub) — implementation count 10 → 11 by ADDITION (like PR #6 nutation / PR #10 traveling_wave); the 2 stubs (parametric, plotting) are unchanged. Pure aggregate_by_genotype(per_plant_df, units) -> (per_genotype_df, per_genotype_units) consumes the PR #14 composed 46-col per-plant frame and emits one row per (plate_id, genotype, treatment) group (plates never pooled — dropna=False so a NaN group key forms its own group, never silently dropped): n_plants_passing_qc, n_plants_excluded, exclusion_reasons (clause→count over excluded plants' qc_failure_reason, _FAILURE_CLAUSE_ORDER-ordered, sentinel-first, clause-incidence), then <trait>_median/<trait>_iqr for 32 numeric traits (NaN-skipping median; IQR = Q75−Q25 via module-qualified scipy.stats.iqr(nan_policy="omit", interpolation="linear"), NaN when < 2 finite — a structural minimum, not a tunable constant), frac_nutating, handedness_mode (most-frequent sign by explicit value counts — NOT Series.mode — tie-break smallest abs then smallest signed) + handedness_consensus_frac. 73-column output. Plants with track_is_clean == False are excluded from every stat (the per-plant track_is_clean/qc_failure_reason already flag them — consumed, not mutated; excluded plants stay auditable by re-filtering the per-plant CSV). Three trait-level decisions: principal_axis_angle (a wrapping circular rad angle) is dropped — a linear median/IQR is unsound near ±π and it is an arbitrary absolute reference direction (circular-statistics summary deferred to a follow-up); helix_signed_area_px2 is aggregated by its magnitude (helix_signed_area_abs_px2_median/_iqr) because its sign is bimodal chirality that a signed median would cancel — direction is carried by handedness_mode/consensus_frac; growth_axis_unreliable is dropped (a track_is_clean failure clause → always False among passers). Other signed traits (v_long_signed*, v_lat_signed*, period_residual_vs_derr_reference) verified per-plant-consistent and aggregated signed. Degenerate-safe: all-excluded group emits a NaN row (counts kept, handedness_mode=0, int dtype preserved via list-of-dicts construction not groupby().apply); empty frame → empty frame with the full typed schema; never raises on degenerate input (the median/IQR helpers pre-filter to finite and short-circuit before any empty/small-slice op, so degenerate groups emit NaN without a RuntimeWarning — no warning suppression needed). Fails loud on a track↔plant 1:1 violation (the literal §7.7 two-level per-plant collapse is deferred to a follow-up; PR #15 raises rather than silently aggregate per-track). New _io.write_per_genotype_csv / read_per_genotype_csv mirror the per-plant pair (CSV + <stem>.units.json + run_metadata.json); the units-coverage check is factored into a shared behavior-preserving _io._validate_units_coverage(df, units, *, fn_name) (the PIPELINE_UNIT_VOCABULARY membership check stays inline in each writer). Pure-pixel (CC-3); no new units-sidecar vocabulary and no new constants (_CONSTANTS_VERSION stays 6). #222 dependency noted: the derived _median/_iqr names inherit the (currently unsuffixed) source names, so #222's unit-suffix rename must treat _median/_iqr as stat-suffixes layered after the unit suffix. Real Nipponbare plate-001 validation (the integration test): 1 passing / 5 excluded (d2_msd_agreement_high:5), all medians finite, all IQRs NaN (n=1), handedness_mode=1, round-tripped through the writer. Reviewed across brainstorm (D1–D8) + 4 /openspec-review rounds (surfacing real bugs reconciled before implementation: wrong test-file targets, the roadmap plant_id-keying contradiction, the run_metadata.json fixed-name clobber, the principal_axis_angle wrapping-angle median bug, the NaN-group-key silent-drop, and the helix_signed_area_px2 chirality-cancellation bug). (parent epic #197; tracking issue child of #197; OpenSpec design.md tracks all reconciliation rounds.)
    • sleap_roots.circumnutation.pipeline.CircumnutationPipeline / compute_traits — circumnutation pipeline composition (PR #14 of the circumnutation program; OpenSpec change add-circumnutation-pipeline). Graduates the pipeline stub into an implementation module (same stub→impl shape as PR #7/#8/#9: implementation count 9 → 10, stub count 3 → 2; callable compute_traits KEEPS its name). Composes the five trait-emitting tiers (Tier 0 kinematics, QC qc, Tier 1 nutation, Tier 2 psi_g, Tier 3c traveling_wave) into one per-plant trait table + CSV + 2 provenance sidecars. No new science — every trait is already computed and tested by its tier. A sequential merge-orchestrator, NOT the per-frame networkx TraitDef DAG the stub docstring/roadmap row described: the circumnutation tiers are per-track DataFrame→DataFrame functions, not per-frame nodes; the documented tier-dependency order (Tier 0 / QC / Tier 1 / Tier 2 independent; Tier 3c depends on Tier 0 + Tier 1) is honored as a fixed call order (deviation recorded in the spec + design). Picklable @attrs.define class with a single constants field; pure compute_traits(inputs, constants=None) -> (per_plant_df, trajectory_df, units_dict) (module-level wrapper preserved) + a separate save(out_path, per_plant_df, units, *, inputs, input_path) (I/O kept out of compute). Composed schema = 46 columns (8 row-identity + Tier 0 10 + QC 10 + Tier 1 8 + Tier 2 4 + Tier 3c 6), built by left-merging each tier projected to [*_IDENTITY_5_TUPLE, *its trait cols] onto the shared per-plant template (so the 3 non-key identity columns can't collide into _x/_y), int64 coercion-with-raise per merge; each tier's dtypes preserved (bool is_nutating, int handedness, string qc_failure_reason). Dedups the Tier 0/Tier 1 recompute PR #10 traveling_wave.compute does redundant-by-design: traveling_wave.compute gains keyword-only tier0_df/tier1_df (both-or-neither; per-frame operand-column validation; the ops-build projects to the operand column so extra columns can't leak); the pipeline passes its already-computed Tier 0/Tier 1 frames → each tier computed once, Tier 3c columns identical (atol=0) to the standalone recompute. Standalone traveling_wave.compute (kwargs omitted) is byte-identical — its 37-test suite + cross-OS canary stay green. growth_axis_unreliable coalesced to the Tier-0-owned column (QC's copy dropped after a Series.equals dtype+value cross-tier equality guard). Provenance — cadence_s/R_px/run_id now captured in run_metadata.json (additive _io.gather_run_metadata(cadence_s=None, R_px=None) kwargs, nullable for existing callers; save threads them from inputs) so a run is reproducible from its sidecars alone — cadence_s determines every period trait and the residual and is otherwise unrecoverable from the .slp. No new constants (_CONSTANTS_VERSION stays 6). Picklable + deterministic (two in-process runs equal as full frames; no separate composed canary — defers to the per-tier canaries). Real Nipponbare plate-001 integration test (compute → save → read-back): 6 plants, the 46-col schema, all 5 tiers present, traveling_wave_residual finite < 0.30 (QPB band), cross-tier growth_axis_unreliable equality, run_metadata["cadence_s"] == 300.0. A trait-emission module (mirroring PR #6 nutation / PR #7 psi_g: compute(trajectory_df, cadence_s, constants=None) -> pd.DataFrame, 5-tuple groupby, 8 row-identity columns, per-plant template merge), and the first consumer of PR #9's spatial-CWT machinery. Division-guarded (v non-finite or lambda_expected ≤ 0 → NaN, no RuntimeWarning); non-nutating tracks NaN the residual via the NaN-gated T_nutation_median while the pure-spatial λ traits stay valid. cgau2 calibration applied in true px via an in-package n-averaged _CGAU2_LAMBDA_CALIBRATION literal (the per-n ratio scatters ~7 % non-monotonically across n∈{200,400,600}, so the consumer averages → a documented ~±5 % systematic; the module never reads tests/data at runtime — wheel-safe; validated against the JSON by a sync test). Extends the PR #9 calibration artifact append-only (capture_spatial_coi_factor.py --append: loads the existing JSON, copies provenance + the 18 rows VERBATIM, measures only the new λ_true knots 100/120/140/150 for all n, appends at the end → +12 rows, existing 18 byte-identical; λ_reported now reaches ~168 px, covering the real plate-001 λ ~142.5 without clamping). No new constants — reuses COI_FRACTION_MAX=0.5 (its first consumer; spatial COI gate coi_valid_fraction < 1−COI_FRACTION_MAX, strict); _CONSTANTS_VERSION stays 6; adds a _TRAVELING_WAVE_TRAIT_UNITS mapping (#222 — px/dimensionless, all in PIPELINE_UNIT_VOCABULARY). Determinism (CC-6): in-process bit-identical (atol=0); the spatial-λ ridge is exact (wavelengths_px[interior], 1:1 with the argmax scale index — tie-flips are discrete, not atol-bounded); the v·T-derived columns inherit the Tier-1 atol=1e-6 canary. DESCOPED — L_gz/L_c-dependent traits + the growth-zone mask (L_gz_estimate, L_c_estimate, B_balance_number, L_gz_steady_state_residual, L_gz_resolvable) are OMITTED (blocked on #230), not reserved as NaN. Real Nipponbare plate-001 validation (NOT synthetic-only, per the PR #7/#8/#9 lesson): the actual compute() on all 6 tracks yields traveling_wave_residual ∈ [0.094, 0.21] (all < 0.30 → QPB holds to ~9–21 %, provisional); a noise-free uniform-λ synthetic recovers λ a priori (47.7 → 49.3 px) AND reads lambda_spatial_variation = 0 (no spurious argmax-quantization floor). Three recorded deviations from theory.md §7.4 (patched this PR, originals preserved in Appendix B(6)): the "bias cancels in the ratio" handoff claim was empirically FALSE (calibrate λ for all consumers); apex_basal_period_consistency (apex-vs-basal, edge-artifact-dominated + orientation-ambiguous) renamed/redefined → lambda_spatial_variation (orientation-robust spread); lambda_spatial_median → _px (pure-pixel, no L_gz mask). Reviewed across brainstorm + a documented real-data investigation + 2 design critical-review rounds + 1 /openspec-review round + 1 focused-science round (surfacing real issues reconciled before/during implementation: a 5-tuple-vs-track_id silent-NaN join bug, a raise-vs-NaN-row crash path, the calibration packaging/wheel bug, the n-dependence systematic, and an over-claimed noise floor). Closes #232 (parent epic #197; see docs/circumnutation/roadmap.md PR #10 row; OpenSpec design.md + docs/superpowers/specs/2026-06-10-add-circumnutation-tier3c-traits-design.md + the investigation report track all reconciliation rounds.)
    • sleap_roots.circumnutation.spatial_cwt — Tier 3b spatial CWT machinery (PR #9 of the circumnutation program; OpenSpec change add-circumnutation-tier3b-spatial-cwt). Graduates the spatial_cwt stub into an implementation module (same stub→impl shape as PR #7/#8: implementation count 7 → 8, stub count 4 → 3; callable compute_scaleogram KEEPS its name, the speculative wavelet=/scale_range= stub kwargs are dropped — derived from constants, mirroring temporal_cwt). A machinery PR mirroring PR #5 temporal_cwt (input-validating, frozen @attrs.define(frozen=True, eq=False) Results, a determinism contract — and no trait emission). Emits 9 trait columns + 1 boolean flag (growth_axis_unreliable) onto the per-plant DataFrame: median per-frame step magnitude (rotation-invariant), signed and absolute medians of the longitudinal/lateral projections (4 columns; rotation-dependent — NaN'd by the gate when net displacement is comparable to per-frame noise), the magnitude-based long_lat_ratio, path_displacement_ratio, peak-to-peak angular_amplitude of the unwrapped velocity-direction time series (per Bastien-Meroz 2016 Eq. 20 / theory.md §3.5), and principal_axis_angle. Pure-pixel + cadence-independent emission: velocity traits are px/frame, downstream user composes convert_to_mm and a future convert_to_per_hour utility for mm/hr output. Adds two shared internal helpers (_noise.compute_sg_residual_xy, _geometry.compute_psi_g) reserved for reuse by PR #3 QC and PR #7 ψ_g respectively. Adds private _io._build_per_plant_template_from_df helper to support tier modules with raw-DataFrame canonical signatures (public build_per_plant_template API unchanged). Theory.md §7.1 expanded from 7 to 9 traits + flag with explicit signed/abs conventions. (parent epic #197; see docs/circumnutation/roadmap.md PR #2 row)
    • New test fixture tests/data/circumnutation_nipponbare_plate_001/ — proofread tracked .slp from the same Nipponbare 0.8% PG MOCK plate that produced the prelim §4.1 numbers (575 frames, 6 tracks, single-node skeleton, ~362 KB Git LFS). Anchors the Tier 0 reference-value sanity test and is reserved for reuse by subsequent tier PRs (#5 Tier 1, #7 Tier 2, etc.).
    • sleap_roots.circumnutation — first OpenSpec change add-circumnutation-foundation (PR #1 of the multi-PR program tracked in docs/circumnutation/roadmap.md). Foundation contracts only — no spectral analysis, no trait emission, no pipeline class. Adds the package skeleton (5 contract modules + 10 stub modules raising NotImplementedError with PR-number references), CircumnutationInputs data class, ROW_IDENTITY_COLUMNS schema, units sidecar JSON, run-metadata sidecar JSON with full provenance fields (git SHA, schema/constants versions, constants snapshot), module-level constants in _constants.py, and the pure-pixel convert_to_mm() downstream-conversion utility. Theory and feasibility docs landed at docs/circumnutation/theory.md, preliminary_results_2026-05-07.md, and roadmap.md. (#198, parent epic #197)
    • TrackedTipPipeline — root-agnostic pipeline that consumes SLEAP-tracked .slp predictions and emits per-track tip-trajectory data plus a minimum-viable substrate of per-track geometric scalars (tip_trajectory_length, tip_displacement_net, tracking_coverage, n_frames_tracked, n_frames_total). Substrate-only by design; velocity / curvature / circumnutation traits live in separate downstream pipelines that reuse this substrate. (#129, Workstream 2 of the 2026-04-23 timelapse design)
    • Series.get_tracked_tips(root_type=None) — long-format DataFrame of per-track tip rows (track_id, frame, tip_x, tip_y), sorted by (track_id, frame). Supports single-node and multi-node skeletons via the [-1] skeleton convention.

    Changed¶

    • requires-python raised from >=3.7 to >=3.10. The previous declaration was stale: the lockfile pulls numpy>=2.x which itself requires Python 3.10+, and CI runs only Python 3.11 (.python-version). Python 3.7 (EOL June 2023), 3.8 (EOL Oct 2024), and 3.9 (EOL Oct 2025) classifiers removed; the project now declares the floor that matches actual support. tool.black.target-version = ["py310"] set explicitly so black uses the 3.10+ parenthesized-context-manager syntax. One pre-existing test (tests/test_viewer.py) was reformatted as a consequence — single multi-context-manager with-statement converted from the line-continuation form to the parenthesized form.
    • Added pywavelets>=1.5 and explicit scipy to runtime dependencies (used by future sleap_roots.circumnutation tier PRs). scipy was previously transitive via scikit-image; the explicit declaration prevents silent breakage.
