## What Changes

- **NEW temporal CWT primitives** — `sleap_roots.circumnutation.temporal_cwt.compute_scaleogram(x, cadence_s, constants=None) -> ScaleogramResult` (canonical signature preserved exactly per the foundation's Package layout) and `sleap_roots.circumnutation.temporal_cwt.extract_ridge(scaleogram_result, constants=None) -> RidgeResult` (free-function sibling matching the `_noise.compute_*` / `_geometry.compute_*` precedent). The stub's forward-looking `wavelet=` and `scale_range=` kwargs are DROPPED. **NOT marked BREAKING:** verified by reading the stub at `sleap_roots/circumnutation/temporal_cwt.py:40` — the stub raises `NotImplementedError` as the FIRST line of the body, before any kwarg access. No production code can have depended on those placeholders (any call previously raised `NotImplementedError` on line 1). All tuning flows via `ConstantsT` instead.
- **NEW `ScaleogramResult` frozen attrs class (design.md D1)** — fields: `scaleogram: complex128 ndarray (n_scales, n_frames)`, `scales: float64 (n_scales,)`, `periods_s: float64 (n_scales,)`, `frequencies_hz: float64 (n_scales,)`, `coi_mask: bool ndarray (n_scales, n_frames)`, `cadence_s: float`, `wavelet: str`.
- **NEW `RidgeResult` frozen attrs class (design.md D2)** — fields: `frame_indices: int64 (n_frames,)`, `periods_s: float64 (n_frames,)`, `amplitudes: float64 (n_frames,)` (`= |C|`), `powers: float64 (n_frames,)` (`= |C|² = amplitudes**2`; redundant by construction, intentionally preserved in the spec scenario), `in_coi: bool (n_frames,)`. Ridge is NOT pre-COI-masked; PR #6's trait emission applies the mask per theory.md §7.2 "COI-masked" language.
- **Wavelet-aware COI mask formula (design.md D3)** — `COI_EFOLDING_FACTOR · scale` half-width per Torrence-Compo's edge-effect e-folding convention, generalized to pywt's cmor parameterization. For cmor1.5-1.0: factor = `√B = √1.5 ≈ 1.225` (empirically verified by step-response measurement across cmor0.5/1.0/1.5/2.0; each matches `√B`). NOT `√2` (Torrence-Compo's ω₀=6 standard Morlet, which is the `B_equivalent=2` special case); NOT `√(2·B) = √3` (round-1 reconciliation's erroneous "doubling for both edges" derivation, refuted in round 2). PR #5's chosen factor establishes the calibration baseline for the QC tier's `coi_fraction_t1` reliability gate (PR #3 reserved the threshold but did not implement the trait).
- **Log-spaced scale axis (design.md D4)** — 64 scales by default. `period_min_s = CWT_PERIOD_MIN_NYQUIST_FACTOR · cadence_s = 2 · cadence_s` (Nyquist). `period_max_s = CWT_PERIOD_MAX_SIGNAL_FRACTION · n_frames · cadence_s = 0.25 · n_frames · cadence_s` (Torrence-Compo n/4 guidance). Scale↔period derivation via `pywt.scale2frequency` round-trip (wavelet-agnostic; correct under `WAVELET_DEFAULT_TEMPORAL` override).
- **Determinism contract (CC-6 / D5)** — two-layer canary: `np.array_equal` for same-process self-consistency (`atol=0`); hardcoded 3-value canary at `[scale_idx_at_target, [3 COI-interior frame indices]]` at `atol=1e-9` matching PR #4's repo baseline. Canary captured against `synthetic.generate_trajectory(random_state=0, n_frames=128, T_nutation_s=3333, cadence_s=300, noise_sigma_px=0)`. `scale_idx_at_target` derived from the public `result.periods_s` field via `int(np.argmin(np.abs(... - 3333.0)))` — NOT a magic int (resilient to scale-count override). Canary purpose: REGRESSION DETECTOR for future pywt / numpy / BLAS drift, not a correctness oracle.
- **Simple per-frame argmax ridge (design.md D6)** — `np.argmax(|scaleogram|, axis=0)` per frame. Deterministic via numpy tie-breaking (smallest index on equal values). Known limitation (Mallat 1999 §4.4.2): per-frame independence can produce spurious jitter at frames where multiple harmonics have similar amplitude; **a follow-up GitHub issue is filed** for PR #6 to add a continuity post-filter (median-window or ridge-following) if `T_nutation_iqr`'s accuracy spec demands it.
- **NEW `_constants.py` additions (design.md D7)** — 4 module-level constants: `COI_EFOLDING_FACTOR = math.sqrt(1.5)`, `CWT_SCALE_COUNT_DEFAULT = 64`, `CWT_PERIOD_MIN_NYQUIST_FACTOR = 2.0`, `CWT_PERIOD_MAX_SIGNAL_FRACTION = 0.25`. `_CONSTANTS_VERSION` bumps `3 → 4`. `ConstantsT` gains 4 new fields. `_default_constants_snapshot()` extended to emit all 4. The existing `NYQUIST_RATIO_MAX` constant gains a docstring cross-reference to `CWT_PERIOD_MAX_SIGNAL_FRACTION` (the two are numerically equal at 0.25 but semantically distinct — flagged explicitly in the spec delta as a scope-bounded internal-docstring touch).
- **Strict validation contract (design.md D8)** — `x` must be 1-D finite ndarray, coerced to float64, `len(x) >= MIN_FRAMES_REQUIRED` derived AT CALL TIME from `int(floor(CWT_PERIOD_MIN_NYQUIST_FACTOR / CWT_PERIOD_MAX_SIGNAL_FRACTION)) + 1` (= 9 at defaults; correctly updates under `ConstantsT` overrides). `cadence_s` accepts Python `int` or `float`, rejects bool/str/complex, must be positive finite (matches PR #4's `_check_float_finite` semantics). Positive-finite guards on `CWT_PERIOD_MAX_SIGNAL_FRACTION` and `CWT_PERIOD_MIN_NYQUIST_FACTOR` close the `SIGNAL_FRACTION=0` ZeroDivisionError edge case. `compute_scaleogram` raises `ValueError`/`TypeError` with the offending field named; `extract_ridge` rejects non-`ScaleogramResult` input with `TypeError`.
