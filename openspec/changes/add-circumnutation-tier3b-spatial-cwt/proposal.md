## Why

PR #9 of the multi-PR circumnutation program (epic #197, roadmap row #9; tracking issue #229) graduates the `spatial_cwt` stub into an implementation module. It is the spatial sibling of PR #5 `temporal_cwt`: input-validating, frozen `attrs` Results, a determinism contract, **no trait emission**. It consumes PR #8's `MidlineResult` (`curvature_px_inv` on the native NON-uniform `arc_length_px` grid), owns the uniform-arc-length resample (the locked PR #8→#9 handoff), runs a `cgau2` spatial Continuous Wavelet Transform of κ(s), and extracts the dominant spatial-wavelength ridge λ(s_a) — the steady-traveling-wave quantity PR #10 needs for `traveling_wave_residual` and `apex_basal_period_consistency`.

**Scope was revised after a pre-spec empirical investigation.** A documented `research:investigate` on the 6 Nipponbare plate-001 proofread tracks established that the theory §7.4 premise — `L_gz` = peak of the |κ(s)| envelope, `L_c` = basal exponential decay (Meroz 2026 Eq. 5–6) — **does not transfer to top-view tip-trail κ(s)**: real trails span ≈3050–3527 px, the growth zone (~29–65 px) is ~1 % of the trail, and the |κ| envelope global-argmax lands basal (48–100 % of span) on every track and every candidate envelope method (Hilbert, cgau2 CWT-power, smoothed-|κ|). Raw κ(s_a) is a roughly steady-amplitude oscillation over the whole trail — the QPB steady-traveling-wave signature (§4.7), exactly as theory §6.5 warned ("the recent trail cannot be inverted… trail-as-midline holds only past the growth zone"). The §7.4 definitions originate from Rivière/Averrhoa whole-organ side-view κ(s,t), not multi-period top-view tip-trail data. PR #9 therefore ships the part the data supports (resample + cgau2 scaleogram + λ_spatial ridge) and **descopes `L_gz`/`L_c` detection** to a tracked follow-up research issue (#230), which blocks the `L_gz`-dependent parts of PR #10 but not `traveling_wave_residual` / `apex_basal_period_consistency`.

## What Changes

- Graduate `spatial_cwt` stub → implementation module (impl 7→8, stub 4→3 in the foundation contract), keeping the locked `compute_scaleogram(kappa, ds, constants=None)` signature (the stub file's `wavelet=`/`scale_range=` kwargs are dropped — derived from `constants`, matching `temporal_cwt`).
- Add 3 public functions: `resample_curvature(...)` (entry; non-uniform→uniform κ(s) resample with sub-noise-mask drop + linear interp + guards), `compute_scaleogram(kappa, ds, constants=None)` (cgau2 spatial CWT), `extract_ridge(scaleogram_result, constants=None)` (per-position dominant λ(s_a) ridge).
- Add 3 frozen `@attrs.define(frozen=True, slots=False, kw_only=True, eq=False)` Result classes: `ResampleResult`, `SpatialScaleogramResult`, `SpatialRidgeResult`.
- Add 3 constants → `_CONSTANTS_VERSION` 5→6: `SPATIAL_COI_EFOLDING_FACTOR` (cgau2 e-folding, empirically measured via a step-response capture script), `CWT_WAVELENGTH_MIN_NYQUIST_FACTOR`, `CWT_WAVELENGTH_MAX_SIGNAL_FRACTION` (spatial siblings of the temporal `CWT_PERIOD_*`). Reuse `CWT_SCALE_COUNT_DEFAULT`, `WAVELET_DEFAULT_SPATIAL="cgau2"`, `NYQUIST_RATIO_MAX`. Snapshot length 35→38.
- Pure-pixel (CC-3): λ/wavelengths in px, curvature/spatial-frequency in px⁻¹. The `px⁻¹` `PIPELINE_UNIT_VOCABULARY` token is deferred to PR #10 (PR #9 emits no sidecar/CSV) — governed by the program-wide unit-suffix convention (#222).
- Determinism (CC-6): no `curve_fit` → the `np.interp` + `pywt.cwt(cgau2)` + argmax-ridge stack holds at `atol=1e-9, rtol=0` cross-OS (like PR #5). Analytic pure-sinusoid-of-known-λ oracle + a captured determinism canary.
- **Recorded theory deviation:** `theory.md` §7.4/§6.3 patched to note the `L_gz`/`L_c` growth-zone-structure traits are not measurable from top-view tip-trail κ(s) as written; λ_spatial is the measurable spatial quantity. Original preserved in Appendix B; changelog "Why" note.
- Foundation-test migration (atomic with the first non-raising commit): `spatial_cwt` moves out of the stub tables into the implementation tables; `_CONSTANTS_VERSION` assertion 5→6.
