## Why

This is PR #2 in the circumnutation analysis program tracked by `docs/circumnutation/roadmap.md`. The foundation (PR #1) shipped contracts: package layout, `CircumnutationInputs`, row-identity schema, units sidecar, run-metadata sidecar, constants, `convert_to_mm`. This PR is the first tier to *emit traits* ‚Äî the **Tier 0 raw kinematic traits** defined in `docs/circumnutation/theory.md` ¬ß7.1. No spectral analysis; no midline reconstruction; just per-track scalar kinematics derived directly from tip positions over time.

Theoretical foundation: `docs/circumnutation/theory.md` ¬ß7.1 (trait list), ¬ß2.1 (coordinate frame), ¬ß2.3 (pure-pixel contract), ¬ß3.5 (`œà_g(t)` definition used for `angular_amplitude`). Empirical anchor: `docs/circumnutation/preliminary_results_2026-05-07.md` ¬ß3.2 / ¬ß3.5 / ¬ß4.1 (Nipponbare plate 001 reference values; **note these are reported as means, not medians ‚Äî the Tier 0 traits emit medians, so the prelim values are order-of-magnitude anchors only**). Cross-cutting concerns addressed: CC-2 (constants ‚Äî uses the foundation's `SG_WINDOW_SHORT`, `SG_DEGREE`, `GROWTH_AXIS_RELIABILITY_K`; `OUTLIER_STEP_RATIO` is reserved for PR #3 QC's `frac_outlier_steps` trait, not used in Tier 0), CC-3 (pure-pixel ‚Äî Tier 0 emits `px/frame` only, time conversion deferred), CC-4 (row identity ‚Äî 8-column schema preserved), CC-5 (growth-axis low-displacement edge case ‚Äî Tier 0 owns the gate; see design.md D2 for the deviation from CC-5 step 3 about which tier emits the flag), CC-9 (logging).

## What Changes

- **NEW Tier 0 trait emission** ‚Äî `sleap_roots.circumnutation.kinematics.compute(trajectory_df, constants=None)` returns a per-plant DataFrame with the 8 row-identity columns plus **10 new columns**: 9 trait columns + 1 boolean flag. Canonical signature preserved exactly as locked in the foundation's Package layout table.
- **NEW trait set (9 traits)**:
  - `v_total_median_px_per_frame` (`px/frame`): median step magnitude.
  - `v_long_signed_median_px_per_frame` (`px/frame`): signed median longitudinal component ‚Äî typically positive for growing plants, ~0 for jitter, negative for retracting.
  - `v_long_abs_median_px_per_frame` (`px/frame`): absolute median longitudinal component.
  - `v_lat_signed_median_px_per_frame` (`px/frame`): signed median lateral ‚Äî ‚âà 0 by symmetry; itself a sanity check.
  - `v_lat_abs_median_px_per_frame` (`px/frame`): absolute median lateral.
  - `long_lat_ratio` (`‚Äî`): `v_long_abs / v_lat_abs`; NaN when denominator is 0.
  - `path_displacement_ratio` (`‚Äî`): `L / D`; NaN only when `D = 0` exactly.
  - `angular_amplitude` (`rad`): `max(œà_g) ‚àí min(œà_g)` where `œà_g = unwrap(atan2(Œîx, Œîy))` per Bastien-Meroz 2016 Eq. 20 / theory.md ¬ß3.5 (argument order is `Œîx` first, then `Œîy` ‚Äî convention-critical for downstream `handedness` sign); rotation-invariant.
  - `principal_axis_angle` (`rad`): `atan2(y_N ‚àí y_1, x_N ‚àí x_1)` ‚Äî full `(‚àíœÄ, œÄ]` range; image-y-downward convention.
- **NEW flag (1 bool column)**: `growth_axis_unreliable` ‚Äî True iff `D < GROWTH_AXIS_RELIABILITY_K (=10) √ó sg_residual_xy_local`. When True, six rotation-dependent traits (`v_long_signed`, `v_long_abs`, `v_lat_signed`, `v_lat_abs`, `long_lat_ratio`, `principal_axis_angle`) are NaN'd; the three rotation-invariant traits (`v_total_median`, `path_displacement_ratio`, `angular_amplitude`) remain valid.
- **NEW helper module** `sleap_roots/circumnutation/_noise.py` ‚Äî `compute_sg_residual_xy(x, y, window, degree) -> float`. Used by Tier 0 (gate value) and reserved for PR #3 (canonical `sg_residual_xy` trait emission). Single source of truth for the formula.
- **NEW helper module** `sleap_roots/circumnutation/_geometry.py` ‚Äî `compute_psi_g(x, y) -> np.ndarray`. Used by Tier 0 (`angular_amplitude`) and reserved for PR #7 (`T_psig_median` CWT input). Returns unwrapped `atan2`-of-velocity per Bastien-Meroz 2016 Eq. 20.
- **REFACTOR `_io.py`** ‚Äî add private `_build_per_plant_template_from_df(df: pd.DataFrame) -> pd.DataFrame` containing the existing drop-duplicates + sort + dtype logic. Public `build_per_plant_template(inputs)` becomes a thin wrapper. No change to its observable behavior or tests. Tier modules with raw-DataFrame canonical signatures (kinematics today; qc, parametric, pipeline later) use the private helper to avoid duplicating the row-identity contract.
- **MODIFIED Package layout requirement** ‚Äî module counts: 7 contract modules (was 5: `__init__.py`, `_constants.py`, `_types.py`, `_io.py`, `units.py`, **plus** `_noise.py`, `_geometry.py`), 1 implementation module (`kinematics` ‚Äî was a stub), 9 remaining stubs (was 10). Stub-table strikes `kinematics`; everything else unchanged.
- **NEW test fixture** `tests/data/circumnutation_nipponbare_plate_001/` ‚Äî direct copy of the prelim ¬ß4.1 analysis source (`plate_001_greyscale.tracked_proofread.slp`, 362 KB, Git LFS) + synthesized `fixture_metadata.csv` + provenance README. Mirrors the existing `tests/data/circumnutation_plate/` (KitaakeX) precedent. Unlocks reference-value sanity tests against the only kinematically characterized plate.
- **UPSTREAM FIX in `Series.get_tracked_tips`** ‚Äî pulled into PR #2 after Nipponbare calibration revealed `Series.get_tracked_tips()` (PR #190) returns BOTH the `PredictedInstance` (tracker's prediction) AND the user-corrected `Instance` for every frame the user proofread, in violation of the function's docstring (*"One row per (track_id, frame)"*). The dual rows propagate `Œîframe=0` divides into every velocity-bearing trait in `kinematics.compute`, causing physically-healthy proofread tracks to fire the growth-axis reliability gate. Fix: refactor `get_tracked_tips` to a two-pass algorithm that drops `PredictedInstance` rows where a user-corrected `Instance` exists for the same `(track_id, frame_idx)`, per `docs/circumnutation/preliminary_results_2026-05-07.md` ¬ß3.1. Same-type duplicates (two `PredictedInstance` or two `Instance` for the same `(track, frame)`) are preserved ‚Äî only the predicted-with-user-correction pattern collapses. New tests in `tests/test_tracked_tip_pipeline.py` ¬ß13.x; spec delta at `openspec/changes/add-circumnutation-tier0-kinematics/specs/tracked-tip-pipeline/spec.md` (MODIFIED Requirement: `Series.get_tracked_tips`).
- **NEW tests** `tests/test_circumnutation_kinematics.py` ‚Äî synthetic exact-value tests (straight-line, pure-noise, circular, NaN-rows, frame-gaps), KitaakeX smoke test (existing fixture), Nipponbare reference-value sanity test (new fixture). Coverage target: 100% on the new modules.
- **DOCS update** `docs/circumnutation/theory.md` ¬ß7.1 ‚Äî expand the trait table from 7 rows to 9 rows with explicit signed/abs convention; annotate that listed `mm/hr` units are *post-conversion*; pipeline emits `px/frame`. Single source of truth for the trait set.
- **CHANGELOG entry** `docs/changelog.md` (lowercase per repo convention).

- **What this change does NOT do:**
  - No `convert_to_per_hour` utility. Per-frame velocities ship as-is; downstream time-axis conversion deferred until a downstream consumer (likely PR #14 pipeline or PR #15 aggregation) demands per-hour output. The orthogonal `convert_to_mm` utility (from PR #1) already handles `px/frame ‚Üí mm/frame` correctly without modification.
  - No `msd_noise_xy` (third independent noise estimator) ‚Äî PR #3 (QC) scope.
  - No manual override of `growth_axis_unreliable` via the metadata CSV ‚Äî researcher-curated veto stays at `qc_cylinder` (per-series) granularity for now; per-track manual override deferred to PR #3 or PR #14.
  - No per-genotype aggregation of these traits ‚Äî PR #15.
  - No Tier 1 / Tier 2 / Tier 3 / Tier 4 trait emission ‚Äî PRs #5‚Äì#11.
  - No spectral analysis. `angular_amplitude` is the only spectral-flavored trait computed here, and it's a simple peak-to-peak extent over the unwrapped velocity-direction time series ‚Äî no CWT, no Fourier.
