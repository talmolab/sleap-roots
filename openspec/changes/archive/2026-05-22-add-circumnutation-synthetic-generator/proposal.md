## Why

Fill in the **synthetic trajectory generator** (PR #4 in the circumnutation program tracked by `docs/circumnutation/roadmap.md`) so downstream tier and aggregation PRs have a deterministic ground-truth fixture builder for Layer-1 validation. Implements `docs/circumnutation/theory.md` §4 (Rivière 2022 Eqs. 1/3/4/5 closed-form realization) + §8 Layer 1 validation strategy.

Theoretical foundation: `theory.md` §3.5 (BM2016 Eq. 20 `atan2(dx, dy)` handedness convention); `theory.md` §4 (Rivière 2022 kinematic model); `theory.md` §4.4 (closed-form `Δφ = 2·ΔL·δ̇₀ / (ωR)`); `theory.md` §8 (Layer 1 tolerances ±5% T, ±15% spatial). Empirical anchor: `docs/circumnutation/preliminary_results_2026-05-07.md` §1, §3.4, §4.1, §4.3 (plate 001 reference values: mean longitudinal step ≈ 4.29 px/frame, T_nutation ≈ 3333 s, peak-to-peak amplitude ≈ 10 px, SG-residual noise σ_xy ≈ 1.83 px, 575 frames @ 5-min cadence). Cross-cutting concerns: **CC-6 (determinism, including across OSs)**, CC-2 (constants — adds 7 defaults + bumps `_CONSTANTS_VERSION`), CC-3 (pure-pixel emission — no `px_per_mm` per foundation spec scenario already locked), CC-9 (logging). Full architectural reasoning lives in `design.md` (D1-D14 + R1-R6 + Migration Plan + Reconciliation Appendix tracking findings from 3 critical-review subagents).

## What Changes

- **NEW synthetic emission** — `sleap_roots.circumnutation.synthetic.generate_trajectory(...)` returns a per-frame `pd.DataFrame` with 8 row-identity columns + `frame` + `tip_x` + `tip_y`, directly consumable by `kinematics.compute` / `qc.compute`. Canonical signature preserved exactly per the foundation's Package layout table (no `px_per_mm`).
- **Parametric closed-form realization (design.md D1)** — Rivière 2022 Eq. 4 realized as `tip_x(t)/tip_y(t) = x0 + v_growth·t·u_g + A_lat·sin(handedness·ω·t)·u_lat + iid_noise`. Closed-form integration uses §4.4 analytical result `Δφ = 2·ΔL·δ̇₀/(ωR)`; literal Eq. 5 PDE integration is explicitly deferred (would couple integration error into Layer-1 tolerances).
- **User-facing aggregate parameters (design.md D2)** — `amplitude_px`, `T_nutation_s`, `growth_rate_px_per_frame`, `noise_sigma_px` instead of the Rivière 6-tuple, because the tip-trajectory only carries 3 aggregate observables (`Δφ`, `v_growth`, `ω`) and 4 of the Rivière params are mathematically inert at tip level. The Rivière correspondence is documented; PR #12 (Layer-1 validation) will wrap with a Rivière-named translation helper once PR #9 / PR #11 land spatial-CWT recovery.
- **xy-quadrature noise interpretation (design.md D1 + critical-review B1)** — `noise_sigma_px` is the xy-quadrature target so QC tier's `sg_residual_xy` recovers `noise_sigma_px` directly (per-axis draws use `σ / √2`). Empirical anchor: plate 001's `sg_residual_xy ≈ 1.83` maps to `noise_sigma_px ≈ 1.83`, not `≈ 2.83`.
- **Determinism contract (CC-6 / D5)** — `random_state: int | np.random.Generator | None`; bit-identical `tip_x` / `tip_y` across runs AND across OSs on 64-bit platforms (citing NEP 19 + numpy PCG64 stability). `noise_sigma_px = 0` short-circuits the RNG draw entirely and preserves caller-supplied Generator state unchanged (D11).
- **NEW `_constants.py` additions** — 7 module-level constants (no `_DEFAULT_` infix per arch I5 + existing convention): `SYNTHETIC_T_NUTATION_S = 3333.0`, `SYNTHETIC_AMPLITUDE_PX = 10.0`, `SYNTHETIC_GROWTH_RATE_PX_PER_FRAME = 4.29`, `SYNTHETIC_NOISE_SIGMA_PX = 2.0`, `SYNTHETIC_CADENCE_S = 300.0`, `SYNTHETIC_N_FRAMES = 575`, `SYNTHETIC_GROWTH_AXIS_ANGLE_RAD = math.pi / 2`. `_CONSTANTS_VERSION` bumps `2 → 3` per the version-sentinel contract. `ConstantsT` gains 7 new fields. `_default_constants_snapshot()` extended to emit all 7. ConstantsT resolution-order locked (D13): explicit kwarg > constants override > module-level default; signature uses `Optional[float] = None` sentinel for the 7 ConstantsT-overridable parameters.
- **NEW spec requirement (1 ADDED)** — Synthetic trajectory generator — covers signature, output schema with explicit dtype contract, closed-form realization, determinism (CC-6), validation, handedness convention (`+1 = CCW` per BM2016 + curl-sign cross-check), and the Rivière correspondence note.
- **MODIFIED Package layout requirement** — synthetic moves from stub → implementation; "8 stub modules" → "7 stub modules"; "2 implementation modules" → "3 implementation modules"; table updated; scenarios "Calling each remaining stub raises NotImplementedError" goes from 8 → 7 modules; new positive-call scenario "`synthetic.generate_trajectory` no longer raises NotImplementedError" mirrors PR #2 / PR #3 precedent.
- **MODIFIED Module-level constants requirement** — extends the set with the 7 new `SYNTHETIC_*` names (no `_DEFAULT_` infix); bumps `_CONSTANTS_VERSION` from `2` to `3`; `_default_constants_snapshot()` and `ConstantsT` extension scenarios updated.
- **NEW tests** — `tests/test_circumnutation_synthetic.py` with 8 sections mirroring PR #3 (2.A schema/structural; 2.B determinism; 2.C parameter recovery via Tier 0 split into exact-equality `amplitude_px=0` and analytical small-angle `amplitude_px=1` and plate-001 sanity `amplitude_px=10`; 2.D round-trip noise sanity via QC; 2.E handedness sign convention + independent curl-sign cross-check; 2.F validation/error parametrized over ~60 invalid input cases; 2.G ConstantsT override + resolution-order; 2.H reference-fixture agreement against `kinematics.compute(plate_001_real_data)` recomputation, NOT hardcoded numbers — per scientific-rigor reviewer B3). 100% coverage on the new `synthetic.py` module.
- **FOUNDATION TEST MIGRATION** — `tests/test_circumnutation_foundation.py` follows PR #3's precedent: remove `("synthetic", "generate_trajectory", 4)` from `STUB_MODULES` (8 → 7 parametrize ids); **split** `STUBS_WITH_CONSTANTS_KWARG` into `STUBS_WITH_CONSTANTS_KWARG` (NotImplementedError-asserting; 5 entries) + new `IMPLEMENTATIONS_WITH_CONSTANTS_KWARG` (no-exception-asserting; 3 entries) — see Migration Plan in design.md. Update `test_schema_version_is_1_and_constants_version_is_2` → `_is_3` and bump the assertion to `_CONSTANTS_VERSION == 3`.
- **DOCS update** — `docs/changelog.md` entry under [Unreleased] / ### Added. No `theory.md` or `roadmap.md` edits are needed in this PR (the existing CC-6 wording already describes the PR #4 contract; roadmap row #4 status update happens post-merge during cleanup).

## Impact

- **Affected specs:** capability `circumnutation` — 2 MODIFIED Requirements (Package layout, Module-level constants) + 1 ADDED Requirement (Synthetic trajectory generator). No other capabilities touched.
- **Affected code:**
  - implementation: `sleap_roots/circumnutation/synthetic.py` (replaces stub with full implementation; new module-level constants/dicts for output schema; helper functions for validation, closed-form math, identity-column construction)
  - extended modules: `sleap_roots/circumnutation/_constants.py` (adds 7 constants + 7 `ConstantsT` fields + version bump + snapshot extension)
  - foundation test migration: `tests/test_circumnutation_foundation.py` (4 surgical edits per PR #3 pattern + the STUBS split-table refactor per arch I3)
  - tests: `tests/test_circumnutation_synthetic.py` (new, 8 sections)
  - docs: `docs/changelog.md` entry
  - no new dependencies; no new test fixtures (2.H reuses the existing PR #2 / PR #3 plate 001 fixture)
- **What this change does NOT do:**
  - **No literal Eq. 5 ODE forward integration.** Closed-form realization per §4.4 instead. Future PR can add ODE integration if needed.
  - **No Rivière-named API (`L_gz_px`, `Delta_L_px`, `delta_dot_0_per_s`, …).** Tip-level degeneracy makes 4 of 6 inputs inert (D2). PR #12 wraps with translation helper when spatial-CWT recovery lands.
  - **No multi-track / `n_tracks` batch parameter.** Single-track per call; callers loop via `seed_seq.spawn(N)` per D7.
  - **No Tier 1+ trait recovery validation.** Layer-1 recovery of `T_nutation`, `L_gz`, `L_c`, `δ̇₀` is deferred to PR #6 / #9 / #11 (recovery tier PRs). PR #4 validates only Tier 0 + QC trait paths, which is the partial Layer-1 status documented in design.md "Layer-1 recovery scope" table.
  - **No pipeline composition of synthetic + tier outputs.** That's PR #14.
  - **No closed-form-plus-numerical-κ-perturbation hybrid** (§4.6 local-contraction wavelet doubling signature). Out of scope.
  - **No `noise_sigma_px = 0` use in Layer-1 validation.** `noise_sigma_px = 0` is for closed-form correctness tests only; PR #12's Layer-1 suite uses `noise_sigma_px > 0` (D11 + scientific-rigor I3).
- **Open scientific blockers:** **none gating this PR.** All blocking findings from the 3 critical-review subagents were reconciled into design.md before scaffolding this proposal (see design.md "Appendix: Critical-Review Reconciliation"). The most consequential reconciliation was the xy-quadrature noise interpretation (B1) and the `growth_rate_px_per_frame = 4.29` calibration (B2).
